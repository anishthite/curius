Algorithm

  1. Aggregate link saves and highlight users by link_id.
  2. Score links with score = 3·savers + 5·highlighters + highlights.
  3. Score repeated highlight text with score = 4·readers + repeats.
  4. Keep newest lists by created_at; keep popular lists by score.

UI layout

  1. Header: short explanation plus buttons for Links/Highlights and popular/newest.
  2. Feed: numbered HN-like entries with title, domain, score, counts, age, and source user when present.
  3. Model note: lightweight clickable formula terms explain why popularity differs from chronology.
  4. Diagram: sparse SVG shows saves and highlights combining into the current ranking.
