Automate static app refresh

main · worktree /Users/anishthite/workspace/curious-index · 2026-07-21 · status shipped · blast low

Summary

Goal
Nightly GitHub Actions refreshes data/curius.sqlite and commits generated apps/**/*.html.
Deploy
Cloudflare Pages direct uploads serve curius-frontpage and curius-analysis.
Persistence
GitHub Actions cache stores ignored SQLite files between runs.
Surface
.github/workflows/update-curius.yml, README.md, URL flags on analysis/build_follower_site.py.
Schema
None.
Open Qs
0.

Algorithm

  1. Restore data/curius.sqlite* from the latest cache key.
  2. Run scraper self-tests and generator self-test.
  3. Run the social graph crawl only when the DB is missing/followless or manual social_refresh is true.
  4. Refresh stale links/highlights with scraper/curius_link_highlight_updater.py.
  5. Build both static apps with deploy URLs, save SQLite cache, and commit generated HTML changes.
  6. Deploy both apps directly when CLOUDFLARE_API_TOKEN exists.

Decisions

IDChoice · Rationale
D-001Use Cloudflare Pages Git integration. It avoids deploy tokens and extra CLI dependencies.
D-002Cache SQLite instead of committing it. The DB stays local/noisy while the generated HTML is reviewable.
D-003Default deploy URLs to curius-frontpage.pages.dev and curius-analysis.pages.dev. Those match the documented Pages project names.
D-004Deploy with direct Pages uploads when a token exists. Wrangler-created projects are not automatically Git-connected.

Tradeoffs

IDPicked over Rejected · Reason
T-001GitHub Actions cache over R2. No credentials are required for first deployment.
T-002Commit generated HTML over building inside Cloudflare. The scraper needs persisted SQLite and long-running Python.
T-003Direct deploy step over dashboard-only Git integration. The created Pages projects work immediately with a token.

Reversibility

R-001 · low. Delete .github/workflows/update-curius.yml, unset Pages Git integration, and rebuild pages locally.

Followups

  • L-001 · Move SQLite persistence to R2 if GitHub cache churn becomes noisy.
  • L-002 · Set CLOUDFLARE_API_TOKEN in GitHub secrets for twice-daily direct deploys.

Definition of done

  • ruby -e "require 'yaml'; YAML.load_file('.github/workflows/update-curius.yml')" parses the workflow.
  • python3 analysis/build_follower_site.py --self-test passes.
  • python3 scraper/curius_link_highlight_updater.py --self-test passes.
  • Generated HTML has no unresolved URL placeholders.
  • Generated relative links resolve within apps/.
  • wrangler pages deploy publishes curius-frontpage and curius-analysis.
  • README names Cloudflare project settings and repo variables.
