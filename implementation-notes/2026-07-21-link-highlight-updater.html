Algorithm

  1. Acquire data/curius_link_highlight_updater.lock.
  2. Seed users from /users/all only when the table is empty.
  3. Pick people whose person_scrapes.completed_at is missing or older than --refresh-hours.
  4. For each picked user, call existing saved-link and highlight refresh methods.
  5. Mark person_scrapes, rebuild site/index.html, and update progress HTML.
  6. In --loop mode, sleep and repeat.

D-002Regenerate only site/index.html. Full graph pages are one-time analysis and expensive to rebuild.
D-003Throttle error rows by updated_at. A permanently failing user should not monopolize every cycle.
