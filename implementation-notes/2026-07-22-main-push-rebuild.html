Decisions

IDChoice · Rationale
D-001Extend the existing workflow. One job already tests, refreshes, rebuilds, commits, and optionally deploys both apps.
D-002Reset to origin/$branch before committing generated apps. It removes stale-run rebase conflicts in generated HTML.
D-003Commit generated apps with [skip ci]. Bot app commits should not trigger another app rebuild.
D-004Skip scraper refresh on push when the SQLite cache exists. Push verification should rebuild pages without waiting on Curius API refreshes.

Tradeoffs

IDPicked over Rejected · Reason
T-001Path-filtered push over a second workflow. It avoids duplicate rebuild definitions.
T-002Reset-and-rebuild over rebase strategy flags. Generated files are cheaper to recreate than merge.
T-003Cached push rebuild over full push refresh. Source-change verification should be fast; scheduled/manual runs own freshness.

Reversibility

R-001 · low. Remove the push trigger and rely on scheduled/manual refreshes.
