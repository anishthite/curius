Algorithm

  1. Compute directed counts, PageRank, weak/strong components, reciprocity, clustering, and undirected core numbers from SQLite.
  2. Place the whole graph with the giant weak component centered and smaller components around it.
  3. Render graph interactions with canvas; render metrics as prose, equations, a small tunable model, glossary, and citations.

UI layout

  1. Graph page: search, minimum-followers filter, view mode, canvas pan/zoom, selected-person reader.
  2. Metrics page: linear narrative, actual metric values, clickable equations, toy graph diagram, glossary, references.

Decisions

IDChoice · Rationale
D-001Use canvas for the graph page. 6,372 nodes and 20,745 edges are too noisy for full-page SVG.
D-002Keep metrics exact except PageRank iteration. Approximate betweenness is omitted because it would add a sampling caveat.
D-003Use lightweight HTML math. The page needs five fixed equations, not a TeX runtime.
D-004Put only the live model before prose on mobile. Dense numbers and glossary follow the narrative.
D-005Shorten graph mode labels. Native select width used the longest option and crowded Fit graph.
D-006Hide α unless PageRank is selected. The slider only affects that model.

T-002one metrics narrative over metric catalogue. The page explains follower flow from edge to rank.
