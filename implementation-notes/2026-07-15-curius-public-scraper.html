API contract

GET /users/all                       -> { users }
GET /users/{userLink}                -> { user: { ..., followingUsers } }
GET /users/{uid}/searchLinks         -> { links }
GET /snippets?uid={uid}&page={page}  -> { highlights: [{ ..., link, user }] }

Algorithm

  1. Fetch /users/all; upsert minimal users.
  2. Fetch every /users/{userLink}; replace that user's outgoing follows.
  3. For each --person, resolve userLink to uid.
  4. Fetch /users/{uid}/searchLinks; replace that user's saved_links.
  5. Fetch /snippets?uid={uid}&page=N until empty; replace that user's highlights.
  6. Write progress HTML after each profile and highlight page.

Decisions

IDChoice · Rationale
D-001Use SQLite. Local scrape needs resumable relational joins without server setup.
D-002Use /users/{userLink} for follows. /users/all omits followingUsers.
D-003Use /snippets?uid= for highlights. It includes the source link object.
D-004Add --all-people with person_scrapes. The all-user crawl needs resume markers.
D-005Serialize object-valued position. Some PDF highlights return coordinate dictionaries instead of integers.
